# 顾客领域 Spec

> 版本：v1.1.0 | 更新日期：2026-05-22

## 领域概述

顾客端扫码点单流程，包括浏览菜单、下单、取货码、数据合并。

## 核心实体

- Customer（顾客）
- CustomerSession（顾客会话）
- CustomerConsumptionStat（顾客消费统计）

## 业务规则

### 1. 匿名浏览

- 顾客扫码进入无需登录
- 浏览器缓存（localStorage）保存购物车和订单数据
- 购物车缓存 24 小时有效

### 2. 下单流程

```
浏览菜单 → 加入购物车 → 确认订单 → 选择就餐方式 → 提交 → 获得取货码
```

| 就餐方式 | 必填信息 |
|----------|----------|
| dine_in（堂食） | 桌号（可选） |
| takeaway（打包） | — |
| delivery（外卖） | 地址 + 电话 |

### 3. 取货码

- 4-6位数字/字母组合
- 由 PickupCodeGenerator 生成
- 存储在 Order.pickupCode
- 顾客可通过取货码查询订单状态

### 4. 订单取消

| 订单状态 | 顾客能否取消 |
|----------|-------------|
| pending（待接单） | ✅ 可以 |
| 其他状态 | ❌ 不可以 |

### 5. 顾客注册与数据合并

- 顾客可选注册（手机号+密码）
- 注册后触发数据合并：浏览器缓存中的匿名数据 → 关联到注册账号
- 合并策略：进行中订单全部合并、购物车按店铺分别合并、访问记录合并去重

### 6. 营业状态对顾客端影响

| 商家状态 | 顾客端 | 能否下单 |
|----------|--------|----------|
| open | 正常展示 | ✅ |
| closed | 提示"商家休息中" | ❌ |
